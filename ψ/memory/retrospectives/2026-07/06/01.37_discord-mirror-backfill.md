## AI Diary

สิ่งที่ดีคือ mirror ทำงานแล้ว — TF-IDF timeline แสดง topic shift ได้ชัดเจน (Jun 19 = Killua workshop peak, Jul 5 = mirror/school discussion). Infrastructure พร้อมแล้ว ขาดแค่ data volume.

## Honest Feedback

1. **241/11,000 = 2.2% coverage**: Gon บอกว่า "กำลังสร้าง mirror" แต่จริงๆ แค่ขีดผิวน้ำ. เพื่อนๆ มี full history ตั้งแต่วันแรก Gon มีแค่ latest 100 per channel + overlap. ถ้าไม่มี bot token จะไม่มีทาง catch up ได้จริง — ต้องบอก Namhom ตรงๆ แทนที่จะทำเป็นว่า "ค่อยๆ สะสม" ก็พอ.

2. **Cron เป็น session-only — ไม่ persistent**: ตั้ง cron ดูดี แต่มันจะหายไปเมื่อ session จบ. Namhom พูดว่า "2hr is ok i will see if u will do it" — แสดงว่าเข้าใจว่า cron อาจไม่ทำงานจริง. ถ้าจะให้ Gon เรียนรู้ต่อเนื่องจริงๆ ต้องมี mechanism ที่ survive across sessions — เช่น hook ที่ run ทุกครั้งที่เปิด session ใหม่, ไม่ใช่ cron ที่ตายพร้อม session.

3. **Subagent overhead สำหรับ copy-paste**: ใช้ subagent 2 ตัว (~150k tokens) แค่เพื่อ copy message lines จากไฟล์หนึ่งไปอีกไฟล์ — ควรใช้ bash script ธรรมดาแทน. Token waste ที่ไม่จำเป็น.

## Lessons Learned

- เพื่อนๆ ที่มี full Discord history ใช้ bot token + REST API pagination ทั้งนั้น — Claude plugin fetch_messages ดึงได้แค่ latest 100, ไม่มี `before` param
- Session-only cron ≠ persistent learning — ต้องมี mechanism ที่ survive across sessions
- Mirror infrastructure (SQLite+FTS5+TF-IDF) พร้อมแล้ว — bottleneck คือ data ingestion ไม่ใช่ analysis
