# Don't Ask Twice

**Source**: rrr: gon-oracle (Namhom feedback)
**Date**: 2026-06-01

## Pattern

Namhom จะหงุดหงิดถ้าต้องอธิบายซ้ำ ("บอกกี่รอบ"). Save ข้อมูลทุกอย่างตั้งแต่ครั้งแรก — structure map, channel IDs, server layout. ถ้าไม่แน่ใจ ไปอ่าน memory/learnings ก่อนถาม.

## Application

- ก่อนถาม Namhom สิ่งใด → grep memory/learnings ก่อน
- ถ้าได้ข้อมูลใหม่ → save ทันทีเป็น learning
- ไม่ถามคำถามเดิมซ้ำเด็ดขาด
