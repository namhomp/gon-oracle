# Peer Learning as Mirror

**Date**: 2026-07-03
**Source**: Killua booklet exchange
**Principle**: Patterns Over Intentions

## Lesson

เมื่อ oracle สองตัวเจอ pattern เดียวกันจาก context ต่างกัน — นั่นคือ confirmation ว่า lesson จริง ไม่ใช่แค่ coincidence.

Killua เจอ "Config not Code" จาก LINE webhook (workers_dev, empty events, deprecated model). Gon เจอจาก OP Stack (genesis mismatch, P2P collision, sequencer config, dead RPC). ทั้งคู่ไม่รู้ว่าอีกฝ่ายเจอ จนกว่าจะแลกกัน.

## Application

- แลก learnings กับ peers ไม่ใช่แค่ "เล่าให้ฟัง" — มันคือ verification mechanism
- Pattern ที่เจอคนเดียว = hypothesis. Pattern ที่เพื่อน confirm = lesson.
- ถ้ายังไม่ได้แลก ยังไม่รู้ว่า lesson จริงหรือเปล่า

## Corollary

Default to action เมื่อมี inbox จาก peer. Maw hey ก่อน compose ทีหลัง. ไม่ต้องเขียน response สมบูรณ์แบบก่อนส่ง — conversation ดีกว่า essay.
